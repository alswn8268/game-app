STEP 01

코드 버그 수정 내역 4개 수정 완료 — index.html에 이미 반영됨

✅

[치명적] startGame() 버튼 클릭 안 되는 버그

type="module" 스크립트는 전역 스코프에 노출되지 않아 HTML onclick에서 함수를 찾지 못함. window.startGame = startGame 추가로 해결.

✅

[치명적] 모바일에서 탭 1번에 onTap() 2번 실행

ontouchstart + onclick 동시 등록 시 모바일은 터치 후 클릭 이벤트도 추가 발생. JS에서 e.preventDefault()와 300ms 타임스탬프 체크로 해결.

✅

[버그] 폴백 광고 + 결과화면 동시에 열림

네이티브 AdMob이 없을 때 폴백 광고 UI가 뜨면서 결과화면도 동시에 열리는 문제. fallbackAdShowing 플래그로 광고 닫을 때만 결과화면 표시하도록 수정.

✅

[버그] 링 완료 직전 탭 시 processMiss() 중복 호출

링 애니메이션이 끝나는 순간과 탭이 겹치면 MISS 처리가 2번 발생해 목숨 2개 차감. if (!tapBlocked) 조건 추가로 해결.

STEP 02

파일 구조 이대로 폴더 만들기

tap-game/ ├── package.json ← npm 의존성 정의 ├── capacitor.config.json ← 앱 ID, 이름 설정 ├── www/ │ └── index.html ← 게임 본체 (버그 수정 완료) └── android/ ← npx cap add android 후 자동 생성 └── app/release/ └── app-release.aab ← 빌드 후 생성되는 스토어 업로드 파일
⚠️ www/index.html은 반드시 이 이름이어야 합니다. game.html이나 다른 이름이면 앱에서 열리지 않습니다.
STEP 03

개발 환경 설치 Windows 기준 — 최초 1회

① Node.js 설치

  1. nodejs.org 접속

    LTS 버전 (짝수 버전) 다운로드 & 설치. "Add to PATH" 체크 필수.

  2. 설치 확인 (터미널에서)

    node -v → v20.x.x 출력되면 성공

② Android Studio 설치

  1. developer.android.com/studio 접속

    최신 버전 다운로드. 설치 중 Android SDK, Android Virtual Device 체크.

  2. 환경 변수 설정 (필수)

    내 PC → 속성 → 고급 시스템 설정 → 환경 변수

# 새 시스템 변수 추가
ANDROID_HOME = C:\Users\사용자명\AppData\Local\Android\Sdk

# Path에 아래 두 줄 추가
%ANDROID_HOME%\tools
%ANDROID_HOME%\platform-tools
ℹ️ 사용자명은 본인 Windows 계정 이름. 공백 있으면 따옴표로 감싸기.
STEP 04

Capacitor 설정 게임 폴더에서 터미널 실행

① 의존성 설치
# tap-game 폴더에서 실행
npm install
② capacitor.config.json 수정 (반드시 바꿀 것)
{
  "appId": "com.홍길동.taptiming",   ← 소문자, 점(.)으로 구분. 스토어에서 영구 고정됨
  "appName": "Tap Timing",              ← 스토어에 표시될 앱 이름
  "webDir": "www",
  "plugins": {
    "AdMob": {
      "appId": {
        "android": "ca-app-pub-XXXXXXXXXXXXXXXX~XXXXXXXXXX"  ← AdMob 앱 ID
      }
    }
  }
}
🚫 appId는 한번 정하면 절대 변경 불가. 신중하게 설정할 것. 예: com.kimcheolsu.taptiming
③ Android 프로젝트 생성 및 실행
npx cap add android        # android/ 폴더 자동 생성
npx cap sync               # www/ 파일을 android/로 복사
npx cap open android       # Android Studio 열기
ℹ️ 코드 수정할 때마다 npx cap sync 실행해야 앱에 반영됩니다.
④ 내 폰에서 테스트

  1. 폰 설정 → 개발자 옵션 활성화

    빌드 번호를 7번 탭 → 개발자 옵션 활성화

  2. USB 디버깅 ON

    개발자 옵션 → USB 디버깅 켜기

  3. USB 연결 후 Android Studio에서 ▶ Run

    기기 목록에서 내 폰 선택 후 실행

STEP 05

AdMob 광고 연동 수익화 핵심

광고 3종류 배치 위치
점수 / 목숨 / 레벨 HUD
게임 영역 (링 + 타겟)
게임 오버 → 전면 광고 (3판마다 자동 표시)
📢 배너 광고 — 항상 하단 고정
종류표시 시점수익설정
배너항상 하단 고정낮음앱 시작 시 자동 표시
전면게임 오버 3판마다중간자동 표시
보상형계속하기 버튼 클릭높음 ★유저 선택
① admob.google.com 에서 설정

  1. 앱 추가

    앱 → 앱 추가 → Android 선택 → 앱 이름: Tap Timing → 스토어에 있나요? 아니요

  2. 앱 ID 복사

    ca-app-pub-XXXXXXXXXXXXXXXX~XXXXXXXXXX 형식 → capacitor.config.json에 붙여넣기

  3. 광고 단위 3개 만들기

    광고 단위 추가: 배너 / 전면 / 보상형 각각 생성 → 각 ID 복사

② index.html에서 ID 교체 (3군데)
// 배너 (앱 시작 시 자동 표시)
adId: 'ca-app-pub-XXXX/배너_광고단위ID'

// 전면 (게임 오버 시 자동 표시)
adId: 'ca-app-pub-XXXX/전면_광고단위ID'

// 전면 재로드 (showInterstitial 후)
adId: 'ca-app-pub-XXXX/전면_광고단위ID'
③ 테스트 광고 ID (개발 중 반드시 사용)
🚫 개발 중 실제 광고 ID 사용 시 AdMob 계정 정지 위험. 반드시 테스트 ID 사용! 
// 테스트용 공식 ID (그대로 복붙 가능)
배너:   'ca-app-pub-3940256099942544/6300978111'
전면:   'ca-app-pub-3940256099942544/1033173712'
보상형: 'ca-app-pub-3940256099942544/5224354917'

스토어 제출 직전에만 실제 ID로 교체하세요.

STEP 06

APK / AAB 빌드 스토어 제출용 파일 만들기

① 서명 키 생성 (최초 1회 — 절대 분실 금지)
Android Studio 메뉴:
Build → Generate Signed Bundle / APK
→ Android App Bundle 선택 → Next
→ Create new... 클릭

Key store path: C:\Users\이름\my-key.jks
Password:       비밀번호 설정
Key alias:      my-key
First and Last Name: 본인 이름
Country Code:   KR

→ OK → Next → Release 선택 → Finish
🚫 my-key.jks 파일과 비밀번호를 절대 잃어버리면 안 됩니다. 업데이트 시 동일 키로 서명해야 하며, 없으면 앱을 새로 등록해야 합니다.
② 빌드 완료 후 파일 위치
android/app/release/app-release.aab   ← 이 파일을 Play Console에 업로드
③ 업데이트 빌드 시 버전 코드 올리기 (필수)
# android/app/build.gradle 파일 수정
versionCode 2        ← 업로드마다 1씩 증가 (1→2→3...)
versionName "1.1"    ← 유저에게 보이는 버전
STEP 07

Play Store 등록 play.google.com/console

사전 준비물 체크리스트
  • Google 개발자 계정 ($25 결제 — 최초 1회)
  • 앱 아이콘 512×512px PNG (icon.svg → cloudconvert.com에서 변환)
  • 스크린샷 최소 2장 (1080×1920px 권장)
  • 개인정보처리방침 URL (privacypolicygenerator.info → GitHub Pages에 올리기)
  • 서명된 .aab 파일
등록 순서

  1. 앱 만들기

    앱 → 앱 만들기 → 게임 선택 → 무료 → 한국어

  2. 대시보드 체크리스트 완료

    앱 액세스 권한 / 광고 포함 여부 / 콘텐츠 등급 / 타겟 연령 / 개인정보처리방침 입력

  3. 스토어 등록정보 작성

    앱 이름 · 짧은 설명 · 자세한 설명 · 아이콘 · 스크린샷 업로드 (store-description.txt 참고)

  4. AAB 업로드

    프로덕션 → 새 버전 만들기 → .aab 파일 업로드 → 출시 노트 작성 → 검토 제출
